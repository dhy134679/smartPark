## 二、各层职责说明

| 层次 | 目录 | 职责 | 依赖关系 |
|------|------|------|---------|
| **路由层** | `routers/` | 接收 HTTP 请求，参数校验，调用服务层，返回响应 | → Services, Schemas |
| **服务层** | `services/` | 核心业务逻辑处理，数据库操作协调 | → Models, Database |
| **模型层** | `models/` | 数据库表映射，ORM 字段定义 | → Database |
| **验证层** | `schemas/` | 请求/响应数据格式验证和序列化 | 无 |
| **AI 引擎层** | `ai/` | 深度学习模型加载、推理、结果后处理 | → PyTorch, 权重文件 |
| **API 封装** | `api/` (前端) | 统一封装后端 API 调用 | → 后端路由层 |

## 三、技术依赖关系

```
Frontend (UniApp/Vue3)
    │
    │ HTTP Request
    ▼
Routers (FastAPI)
    │
    │ 调用
    ▼
Services (业务逻辑) ←──────→ AI Engine (PyTorch)
    │                         │
    │ ORM 操作                │ 模型推理
    ▼                         ▼
Models (SQLAlchemy) ──────→ DB (MySQL 8.0)
                     PyTorch → weights/*.pth
```

## 四、开发规范

### 4.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件名 | 小写下划线 | `parking_record.py` |
| 类名 | 大驼峰 | `ParkingRecord` |
| 函数名 | 小写下划线 | `get_free_spots()` |
| 常量 | 全大写下划线 | `MAX_DAILY_FEE` |
| API 路径 | 小写连字符/下划线 | `/api/parking/entry` |
| 数据库表名 | 小写下划线复数 | `parking_records` |
| 数据库字段 | 小写下划线 | `plate_number` |
| Vue 组件 | 大驼峰 | `ParkingMap.vue` |

### 4.2 代码注释规范

- 所有注释使用**中文**
- 函数/类必须有 docstring 说明功能和参数
- 复杂逻辑需要行内注释解释

### 4.3 API 响应规范

```json
// 成功响应
{
    "code": 200,
    "message": "操作成功",
    "data": { ... }
}

// 错误响应
{
    "code": 400,
    "message": "参数错误：手机号格式不正确",
    "data": null
}
```

