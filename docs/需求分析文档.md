# 基于微信小程序的智能停车场系统 —— 需求分析文档

## 一、项目背景与问题分析

### 1.1 背景描述

随着城市化进程加速和居民汽车保有量的快速增长，老旧小区停车难问题日益严峻。由于历史规划不足，老旧小区车位配比远低于现行标准（通常不足 1:0.5），车位数量严重不足，停车资源紧张。

### 1.2 现状问题

| 问题 | 描述 |
|------|------|
| 车位供需失衡 | 老旧小区车位数远低于住户车辆数，高峰时段一位难求 |
| 管理手段落后 | 多数小区仍靠人工巡查、手动登记，效率低下 |
| 车位利用率低 | 部分业主白天外出，车位闲置却无法被他人使用 |
| 外来车辆管理混乱 | 缺少有效的进出管控和计费手段，外来车辆长期占用 |
| 停车体验差 | 车主进入小区后需盲目寻找车位，浪费时间且增加尾气排放 |

### 1.3 解决思路

设计一个**基于微信小程序的智能停车场管理系统**，综合运用深度学习、计算机视觉、时序预测等技术，实现停车资源的智能化管理和高效利用。

## 二、系统用户分析

### 2.1 用户角色

| 角色 | 说明 | 主要操作 |
|------|------|---------|
| **小区业主** | 拥有小区车位的住户 | 上传车位空闲信息、查看停车记录、绑定车辆、查看导航 |
| **外来车主** | 临时停靠的非小区车辆驾驶者 | 查看空闲车位、停车导航、查看费用、模拟支付 |
| **系统管理员** | 停车场管理人员 | 管理车位信息、查看收费统计、配置计费规则、查看车辆进出记录 |

## 三、功能性需求

### 3.1 用户管理模块

| 编号 | 功能需求 | 优先级 | 描述 |
|------|---------|--------|------|
| F-01 | 用户注册 | 高 | 支持手机号注册，设置密码和基本信息 |
| F-02 | 用户登录 | 高 | 手机号 + 密码登录，返回 JWT 令牌 |
| F-03 | 个人信息管理 | 中 | 查看和修改个人信息 |
| F-04 | 车辆绑定 | 高 | 绑定车牌号码，系统自动标记为小区车辆 |

### 3.2 车位管理模块

| 编号 | 功能需求 | 优先级 | 描述 |
|------|---------|--------|------|
| F-05 | 车位信息展示 | 高 | 实时展示所有车位状态（空闲/占用/预留） |
| F-06 | 车位空闲上传 | 高 | 车主标记自己车位为"空闲可共享"，设置共享时间段 |
| F-07 | 空闲车位统计 | 高 | 首页显示当前小区总车位数、空闲数、占用数 |
| F-08 | 车位状态实时更新 | 中 | 车位状态变化时通过 WebSocket 实时推送到小程序 |

### 3.3 车牌识别模块

| 编号 | 功能需求 | 优先级 | 描述 |
|------|---------|--------|------|
| F-09 | 车牌图像上传 | 高 | 支持拍照或从相册选择车辆图片 |
| F-10 | 车牌自动检测 | 高 | YOLOv5 模型自动检测图片中的车牌区域 |
| F-11 | 车牌字符识别 | 高 | LPRNet 模型识别车牌字符，输出完整车牌号 |
| F-12 | 小区车辆判别 | 高 | 将识别结果与白名单比对，判断是否小区车辆 |
| F-13 | 识别结果展示 | 中 | 展示检测框、车牌号、置信度、车辆类型 |

### 3.4 出入管理与模拟模块

| 编号 | 功能需求 | 优先级 | 描述 |
|------|---------|--------|------|
| F-14 | 车辆入场模拟 | 高 | 上传图片 → 识别车牌 → 判断类型 → 分配车位 → 开始计时 |
| F-15 | 车辆出场模拟 | 高 | 上传图片 → 识别车牌 → 计算费用 → 模拟支付 → 释放车位 |
| F-16 | 完整流程可视化 | 中 | 在小程序中完整展示入场-停车-出场的全流程动画/步骤 |
| F-17 | 车辆进出计数 | 中 | 实时统计当日入场/出场车辆数 |

### 3.5 计费管理模块

| 编号 | 功能需求 | 优先级 | 描述 |
|------|---------|--------|------|
| F-18 | 停车时长计算 | 高 | 根据入场和出场时间自动计算停车时长 |
| F-19 | 费用计算 | 高 | 按配置规则计算费用（首30分钟免费、每小时5元、每日封顶50元） |
| F-20 | 小区车辆免费 | 高 | 已注册的小区车辆停车免费 |
| F-21 | 模拟支付 | 中 | 展示支付界面和支付流程（不接入真实支付） |
| F-22 | 收费记录查询 | 中 | 查看历史停车记录和收费明细 |

### 3.6 停车导航模块

| 编号 | 功能需求 | 优先级 | 描述 |
|------|---------|--------|------|
| F-23 | 停车场平面图展示 | 高 | 通过 Canvas 绘制停车场内部平面图 |
| F-24 | 空闲车位标注 | 高 | 在平面图上用不同颜色标注空闲/占用车位 |
| F-25 | 路径规划 | 中 | A* 算法计算从入口到目标空闲车位的最优路径 |
| F-26 | 导航路线绘制 | 中 | 在 Canvas 上绘制行驶路线引导车主 |

### 3.7 车位预测模块

| 编号 | 功能需求 | 优先级 | 描述 |
|------|---------|--------|------|
| F-27 | 车位满时提示 | 高 | 当所有车位占满时，在小程序上弹出提示 |
| F-28 | 空闲时间预测 | 高 | LSTM 模型预测下一个车位可能空出的时间 |
| F-29 | 预测结果展示 | 中 | 图表化展示预测的车位占用率趋势 |
| F-30 | 历史数据采集 | 中 | 自动记录车位状态变化日志，作为训练数据 |

## 四、非功能性需求

| 编号 | 需求类型 | 描述 |
|------|---------|------|
| NF-01 | 性能 | 车牌识别响应时间 < 3 秒（含网络传输） |
| NF-02 | 性能 | 车位预测计算响应时间 < 2 秒 |
| NF-03 | 性能 | API 接口平均响应时间 < 500ms |
| NF-04 | 可用性 | 界面简洁直观，操作流程不超过 3 步 |
| NF-05 | 兼容性 | 兼容微信小程序和 H5 浏览器 |
| NF-06 | 安全性 | 密码加密存储，接口 JWT 认证 |
| NF-07 | 可维护性 | 代码结构清晰，模块化设计 |
| NF-08 | 可扩展性 | 预留接口便于后续接入真实硬件设备 |

```

      │

      │
      ▼

      │
      ▼
查询白名单数据库 ──────── 是小区车辆？
      │                        │
      │ 否                     │ 是
      ▼                        ▼
创建停车记录             创建停车记录
开始计时收费             标记为免费
      │                        │
      ▼                        ▼
      ├────────────────────────┤
      │                        │
      ▼                        ▼
查询空闲车位 ──────── 有空闲？
      │                   │
      │ 否                │ 是
      ▼                   ▼
显示满位提示         A* 路径规划
预测空闲时间         显示导航路线
      │                   │
      ▼                   ▼
```

```
      ▼
```

## 七、数据字典

### 7.1 数据项定义

| 数据项名 | 含义 | 类型 | 长度 | 取值范围 |
|----------|------|------|------|---------|
| 用户ID | 用户唯一标识 | INT | - | 自增 |
| 手机号 | 用户注册手机号 | VARCHAR | 20 | 11位手机号 |
| 车牌号 | 车牌号码 | VARCHAR | 20 | 如"京A12345" |
| 车位编号 | 车位唯一标识 | VARCHAR | 10 | 如"A-01" |
| 区域 | 停车区域 | VARCHAR | 5 | A/B/C |
| 车位状态 | 当前状态 | VARCHAR | 20 | free/occupied/reserved |
| 入场时间 | 车辆入场时间 | DATETIME | - | - |
| 出场时间 | 车辆出场时间 | DATETIME | - | - |
| 停车时长 | 停靠总时长 | INT | - | 分钟 |
| 停车费用 | 计算所得费用 | DECIMAL | 10,2 | 0.00 ~ 9999.99 |
| 占用率 | 车位占用比例 | FLOAT | - | 0.0 ~ 1.0 |

### 7.2 数据流定义

| 数据流名 | 组成 | 来源 | 去向 |
|----------|------|------|------|
| 车牌图片 | 图像文件 | 用户上传 | 车牌识别处理 |
| 识别结果 | 车牌号 + 置信度 + 车辆类型 | 车牌识别处理 | 出入管理处理 |
| 车位信息 | 车位编号 + 状态 + 坐标 | 车位管理处理 | 导航规划 / 前端展示 |
| 停车记录 | 车牌号 + 入场时间 + 出场时间 + 费用 | 计费处理 | 数据库存储 |
| 导航路线 | 路径节点序列 | 导航规划处理 | 前端展示 |
| 预测结果 | 预测时间 + 占用率趋势 | 车位预测处理 | 前端展示 |
