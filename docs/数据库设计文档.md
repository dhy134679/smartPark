# 基于微信小程序的智能停车场系统 —— 数据库设计文档

## 一、数据库概述

### 1.1 数据库选型

本系统采用 **MySQL 8.0** 作为关系型数据库管理系统。

| 项目 | 说明 |
|------|------|
| 数据库名称 | `smart_parking` |
| 字符集 | `utf8mb4` |
| 排序规则 | `utf8mb4_general_ci` |
| 存储引擎 | InnoDB（支持事务和外键） |

### 1.2 创建数据库

```sql
CREATE DATABASE IF NOT EXISTS smart_parking
DEFAULT CHARACTER SET utf8mb4
DEFAULT COLLATE utf8mb4_general_ci;

USE smart_parking;
```

**实体关系说明**：
- 一个 **用户** 可以绑定多辆 **车辆**（1:N）
- 一个 **用户** 可以拥有多个 **车位**（1:N）
- 一辆 **车辆** 可以产生多条 **停车记录**（1:N）
- 一个 **车位** 可以产生多条 **停车记录**（1:N）
- 一个 **车位** 可以产生多条 **状态日志**（1:N）
- **收费规则** 为独立配置表

## 三、数据表详细设计

### 3.1 用户表（users）

**功能**：存储系统用户信息，包括小区业主和管理员。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|---------|------|--------|------|
| `id` | INT | PRIMARY KEY, AUTO_INCREMENT | - | 用户唯一标识 |
| `phone` | VARCHAR(20) | UNIQUE, NOT NULL | - | 手机号（登录账号） |
| `name` | VARCHAR(50) | NOT NULL | - | 用户姓名 |
| `password_hash` | VARCHAR(128) | NOT NULL | - | 密码哈希值（bcrypt） |
| `role` | VARCHAR(20) | NOT NULL | 'resident' | 角色：resident（业主）、admin（管理员） |
| `is_resident` | TINYINT(1) | NOT NULL | 1 | 是否为小区业主：1=是，0=否 |
| `created_at` | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

```sql
CREATE TABLE users (
    id          INT AUTO_INCREMENT PRIMARY KEY COMMENT '用户ID',
    phone       VARCHAR(20)  NOT NULL UNIQUE COMMENT '手机号',
    name        VARCHAR(50)  NOT NULL COMMENT '姓名',
    password_hash VARCHAR(128) NOT NULL COMMENT '密码哈希',
    role        VARCHAR(20)  NOT NULL DEFAULT 'resident' COMMENT '角色：resident/admin',
    is_resident TINYINT(1)   NOT NULL DEFAULT 1 COMMENT '是否小区业主',
    created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_phone (phone),
    INDEX idx_role (role)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

---

### 3.2 车辆表（vehicles）

**功能**：存储车辆基本信息及其与车主的关联关系。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|---------|------|--------|------|
| `id` | INT | PRIMARY KEY, AUTO_INCREMENT | - | 车辆唯一标识 |
| `plate_number` | VARCHAR(20) | UNIQUE, NOT NULL | - | 车牌号码（如"京A12345"） |
| `owner_id` | INT | FOREIGN KEY → users.id | NULL | 车主用户ID |
| `brand` | VARCHAR(50) | NULL | NULL | 车辆品牌 |
| `color` | VARCHAR(20) | NULL | NULL | 车辆颜色 |
| `vehicle_type` | VARCHAR(20) | NOT NULL | 'car' | 车辆类型：car/suv/truck |
| `is_resident` | TINYINT(1) | NOT NULL | 0 | 是否小区注册车辆 |
| `is_active` | TINYINT(1) | NOT NULL | 1 | 是否启用 |
| `created_at` | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |

```sql
CREATE TABLE vehicles (
    id            INT AUTO_INCREMENT PRIMARY KEY COMMENT '车辆ID',
    plate_number  VARCHAR(20)  NOT NULL UNIQUE COMMENT '车牌号码',
    owner_id      INT          NULL COMMENT '车主ID',
    brand         VARCHAR(50)  NULL COMMENT '车辆品牌',
    color         VARCHAR(20)  NULL COMMENT '车辆颜色',
    vehicle_type  VARCHAR(20)  NOT NULL DEFAULT 'car' COMMENT '车辆类型',
    is_resident   TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '是否小区车辆',
    is_active     TINYINT(1)   NOT NULL DEFAULT 1 COMMENT '是否启用',
    created_at    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    FOREIGN KEY (owner_id) REFERENCES users(id) ON DELETE SET NULL,
    INDEX idx_plate (plate_number),
    INDEX idx_owner (owner_id),
    INDEX idx_resident (is_resident)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='车辆表';
```

### 3.3 车位表（parking_spots）

**功能**：存储停车场车位信息，包含位置坐标（用于导航）和共享状态。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|---------|------|--------|------|
| `id` | INT | PRIMARY KEY, AUTO_INCREMENT | - | 车位唯一标识 |
| `spot_number` | VARCHAR(10) | UNIQUE, NOT NULL | - | 车位编号（如"A-01"） |
| `zone` | CHAR(1) | NOT NULL | - | 所属区域：A/B/C |
| `status` | VARCHAR(20) | NOT NULL | 'free' | 状态：free/occupied/reserved |
| `owner_id` | INT | FOREIGN KEY → users.id | NULL | 车位所有者ID |
| `is_shared` | TINYINT(1) | NOT NULL | 0 | 是否共享 |
| `shared_start` | DATETIME | NULL | NULL | 共享开始时间 |
| `shared_end` | DATETIME | NULL | NULL | 共享结束时间 |
| `x_pos` | FLOAT | NOT NULL | 0 | 车位X坐标（导航用） |
| `y_pos` | FLOAT | NOT NULL | 0 | 车位Y坐标（导航用） |
| `created_at` | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |

```sql
CREATE TABLE parking_spots (
    id           INT AUTO_INCREMENT PRIMARY KEY COMMENT '车位ID',
    spot_number  VARCHAR(10)  NOT NULL UNIQUE COMMENT '车位编号',
    zone         CHAR(1)      NOT NULL COMMENT '区域：A/B/C',
    status       VARCHAR(20)  NOT NULL DEFAULT 'free' COMMENT '状态：free/occupied/reserved',
    owner_id     INT          NULL COMMENT '车位所有者',
    is_shared    TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '是否共享',
    shared_start DATETIME     NULL COMMENT '共享开始时间',
    shared_end   DATETIME     NULL COMMENT '共享结束时间',
    x_pos        FLOAT        NOT NULL DEFAULT 0 COMMENT 'X坐标',
    y_pos        FLOAT        NOT NULL DEFAULT 0 COMMENT 'Y坐标',
    created_at   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    FOREIGN KEY (owner_id) REFERENCES users(id) ON DELETE SET NULL,
    INDEX idx_zone (zone),
    INDEX idx_status (status),
    INDEX idx_owner (owner_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='车位表';
```

### 3.4 停车记录表（parking_records）

**功能**：记录每次车辆出入场的完整信息，包含计时和计费数据。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|---------|------|--------|------|
| `id` | INT | PRIMARY KEY, AUTO_INCREMENT | - | 记录唯一标识 |
| `vehicle_id` | INT | FOREIGN KEY → vehicles.id | - | 车辆ID |
| `spot_id` | INT | FOREIGN KEY → parking_spots.id | NULL | 停放车位ID |
| `plate_number` | VARCHAR(20) | NOT NULL | - | 车牌号码（冗余字段便于查询） |
| `entry_time` | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 入场时间 |
| `exit_time` | DATETIME | NULL | NULL | 出场时间 |
| `duration_minutes` | INT | NULL | NULL | 停车时长（分钟） |
| `fee` | DECIMAL(10,2) | NOT NULL | 0.00 | 停车费用（元） |
| `is_resident` | TINYINT(1) | NOT NULL | 0 | 是否小区车辆 |
| `is_paid` | TINYINT(1) | NOT NULL | 0 | 是否已支付 |
| `payment_time` | DATETIME | NULL | NULL | 支付时间 |
| `status` | VARCHAR(20) | NOT NULL | 'parked' | 状态：parked/exited/paid |
| `entry_image` | VARCHAR(255) | NULL | NULL | 入场图片路径 |
| `exit_image` | VARCHAR(255) | NULL | NULL | 出场图片路径 |
| `created_at` | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |

```sql
CREATE TABLE parking_records (
    id               INT AUTO_INCREMENT PRIMARY KEY COMMENT '记录ID',
    vehicle_id       INT           NOT NULL COMMENT '车辆ID',
    spot_id          INT           NULL COMMENT '车位ID',
    plate_number     VARCHAR(20)   NOT NULL COMMENT '车牌号码',
    entry_time       DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '入场时间',
    exit_time        DATETIME      NULL COMMENT '出场时间',
    duration_minutes INT           NULL COMMENT '停车时长(分钟)',
    fee              DECIMAL(10,2) NOT NULL DEFAULT 0.00 COMMENT '停车费用',
    is_resident      TINYINT(1)    NOT NULL DEFAULT 0 COMMENT '是否小区车辆',
    is_paid          TINYINT(1)    NOT NULL DEFAULT 0 COMMENT '是否已支付',
    payment_time     DATETIME      NULL COMMENT '支付时间',
    status           VARCHAR(20)   NOT NULL DEFAULT 'parked' COMMENT '状态：parked/exited/paid',
    entry_image      VARCHAR(255)  NULL COMMENT '入场图片路径',
    exit_image       VARCHAR(255)  NULL COMMENT '出场图片路径',
    created_at       DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    FOREIGN KEY (vehicle_id) REFERENCES vehicles(id) ON DELETE CASCADE,
    FOREIGN KEY (spot_id) REFERENCES parking_spots(id) ON DELETE SET NULL,
    INDEX idx_plate (plate_number),
    INDEX idx_entry_time (entry_time),
    INDEX idx_status (status),
    INDEX idx_vehicle (vehicle_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='停车记录表';
```

### 3.5 收费规则表（fee_rules）

**功能**：配置停车计费规则，支持多套规则切换。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|---------|------|--------|------|
| `id` | INT | PRIMARY KEY, AUTO_INCREMENT | - | 规则ID |
| `name` | VARCHAR(50) | NOT NULL | - | 规则名称 |
| `free_minutes` | INT | NOT NULL | 30 | 免费时长（分钟） |
| `rate_per_hour` | DECIMAL(10,2) | NOT NULL | 5.00 | 每小时收费（元） |
| `max_daily` | DECIMAL(10,2) | NOT NULL | 50.00 | 每日最高收费（元） |
| `is_active` | TINYINT(1) | NOT NULL | 1 | 是否启用 |
| `created_at` | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |

```sql
CREATE TABLE fee_rules (
    id            INT AUTO_INCREMENT PRIMARY KEY COMMENT '规则ID',
    name          VARCHAR(50)   NOT NULL COMMENT '规则名称',
    free_minutes  INT           NOT NULL DEFAULT 30 COMMENT '免费时长(分钟)',
    rate_per_hour DECIMAL(10,2) NOT NULL DEFAULT 5.00 COMMENT '每小时收费',
    max_daily     DECIMAL(10,2) NOT NULL DEFAULT 50.00 COMMENT '每日封顶',
    is_active     TINYINT(1)    NOT NULL DEFAULT 1 COMMENT '是否启用',
    created_at    DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='收费规则表';
```

### 3.6 车位状态日志表（spot_status_logs）

**功能**：记录车位状态变化历史，作为 LSTM 预测模型的训练数据源。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|---------|------|--------|------|
| `id` | INT | PRIMARY KEY, AUTO_INCREMENT | - | 日志ID |
| `spot_id` | INT | FOREIGN KEY → parking_spots.id | - | 关联车位ID |
| `status` | VARCHAR(20) | NOT NULL | - | 变更后状态 |
| `occupancy_rate` | FLOAT | NOT NULL | 0.0 | 当前全局车位占用率（0~1） |
| `total_occupied` | INT | NOT NULL | 0 | 当前总占用车位数 |
| `total_free` | INT | NOT NULL | 0 | 当前总空闲车位数 |
| `hour` | TINYINT | NOT NULL | 0 | 小时（0-23） |
| `day_of_week` | TINYINT | NOT NULL | 0 | 星期几（0=周一，6=周日） |
| `is_weekend` | TINYINT(1) | NOT NULL | 0 | 是否周末 |
| `timestamp` | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 记录时间 |

```sql
CREATE TABLE spot_status_logs (
    id              INT AUTO_INCREMENT PRIMARY KEY COMMENT '日志ID',
    spot_id         INT          NOT NULL COMMENT '车位ID',
    status          VARCHAR(20)  NOT NULL COMMENT '状态',
    occupancy_rate  FLOAT        NOT NULL DEFAULT 0.0 COMMENT '占用率',
    total_occupied  INT          NOT NULL DEFAULT 0 COMMENT '占用数',
    total_free      INT          NOT NULL DEFAULT 0 COMMENT '空闲数',
    hour            TINYINT      NOT NULL DEFAULT 0 COMMENT '小时(0-23)',
    day_of_week     TINYINT      NOT NULL DEFAULT 0 COMMENT '星期几(0-6)',
    is_weekend      TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '是否周末',
    timestamp       DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '记录时间',
    FOREIGN KEY (spot_id) REFERENCES parking_spots(id) ON DELETE CASCADE,
    INDEX idx_spot (spot_id),
    INDEX idx_timestamp (timestamp),
    INDEX idx_hour (hour),
    INDEX idx_day (day_of_week)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='车位状态日志表（LSTM训练数据）';
```

## 六、数据完整性约束

| 约束类型 | 说明 |
|---------|------|
| 实体完整性 | 每张表使用 INT AUTO_INCREMENT 作为主键 |
| 参照完整性 | 外键约束保证表间引用一致性（如 vehicle.owner_id → users.id） |
| 域完整性 | 通过 NOT NULL、DEFAULT、数据类型限定字段取值范围 |
| 用户定义完整性 | 手机号 UNIQUE 约束、车牌号 UNIQUE 约束保证业务唯一性 |
| 级联策略 | 车主删除时车辆 owner_id 置 NULL；车辆删除时停车记录级联删除 |
